# DOREA NETWORK — CONTRACTS.md

## Purpose

This document defines the contract system for Dorea Network.

Contracts are the behavioral foundation of the runtime.

Contracts define:
- how moments execute
- how communities respond
- how continuity of care is preserved
- how governance policies operationalize behavior
- how delegated participation authority functions
- how agents coordinate actions
- how runtime orchestration occurs
- how continuity remains available throughout daily life
- how requests are acknowledged and preserved relationally

This backend is intentionally dedicated to:
- the Dorea Network operational domain
- Dorea moments of grace
- Dorea continuity of care
- Dorea invitation-based trust communities

This contract architecture is NOT intended to become:
- a generalized workflow engine
- a reusable orchestration framework
- a multi-tenant contract platform
- a generalized automation system

Generalized runtime concepts exist only to support:
- Dorea operational continuity

Contracts are:
- governed
- versioned
- observable
- replayable
- executable

Contracts are NOT:
- static workflows
- hardcoded business logic
- UI behavior
- hidden automation

If implementation conflicts with this document:
- THIS DOCUMENT IS CORRECT
