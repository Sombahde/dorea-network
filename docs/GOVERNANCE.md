The runtime does not govern the community.

The community governs the runtime.

The Dorea backend exists specifically to support:
- Dorea Network
- Dorea moments of grace
- Dorea continuity of care
- Dorea invitation-based communities

The runtime is intentionally:
- Dorea-specific
- grace-oriented
- continuity-centered
- governance-driven

- a generic workflow engine
- a reusable community operating system
- a multi-tenant SaaS governance framework

Generalized runtime concepts exist only to support:
- Dorea operational continuity

Governance exists to:
- define behavioral expectations
- establish participation boundaries
- preserve trust relationships
- guide escalation behavior
- protect vulnerable members
- coordinate continuity of care
- authorize delegated runtime behavior

The runtime operationalizes the governance defined by the community.
