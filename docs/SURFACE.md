# DOREA NETWORK — SURFACE.md

## Purpose

This document defines the surface architecture and user experience philosophy for Dorea Network.

This document governs:
- mobile experiences
- browser experiences
- tablet experiences
- rendering philosophy
- interaction patterns
- surface behavior
- language behavior
- UX constraints

This document intentionally limits Dorea surfaces to:
- mobile
- browser
- tablet

This document does NOT define:
- backend orchestration
- runtime logic
- OODA behavior
- session management

Those belong to:
- ARCHITECTURE.md
- SESSION.md
- RUNTIME_RULES.md

If implementation conflicts with this document:
- THIS DOCUMENT IS CORRECT

---

# CORE SURFACE PRINCIPLE

The frontend is:
- a rendering surface
- a transport layer
- a relational experience

The frontend is NOT:
- the runtime
- the orchestrator
- the session authority
- the decision engine

The runtime owns:
- orchestration
- progression
- contracts
- decisions
- telemetry
- continuity

The surface renders runtime state.
