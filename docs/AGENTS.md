# DOREA NETWORK — AGENTS.md

## Purpose

This document defines the agent architecture for Dorea Network.

Agents are governed runtime participants responsible for coordinating bounded moments of grace within the Dorea runtime.

Agents are:
- runtime-controlled
- contract-governed
- telemetry-observable
- session-aware
- replayable
- bounded

Agents are NOT:
- autonomous actors
- independent orchestration engines
- unrestricted AI systems
- generalized assistants
- self-governing entities

The Dorea agent architecture is intentionally dedicated to:
- the Dorea Network operational domain
- Dorea moments of grace
- Dorea continuity of care
- Dorea participation coordination
- Dorea governance visibility

This architecture is NOT intended to become:
- a generalized multi-agent platform
- an autonomous AI ecosystem
- an unrestricted orchestration framework
- a generalized chatbot architecture

Generalized agent concepts exist only to support:
- Dorea operational continuity

If implementation conflicts with this document:
- THIS DOCUMENT IS CORRECT
